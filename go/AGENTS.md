# Agent guide: Go implementation

Scoped notes for `go/`. See the [root AGENTS.md](../AGENTS.md) for the rules that
apply across both implementations. This is a **port** of the canonical TypeScript
plugin — match its observable behaviour; do not invent new behaviour here.

## Commands

```sh
go build ./...
go test ./...
go vet ./...
```

To build and test against the Jsonic GitHub `main` branch:

```sh
go get github.com/jsonicjs/jsonic/go@main
```

The Go submodule is versioned with `go/vX.Y.Z` tags; `@main` resolves to the
latest such commit. The only required dependency is the Jsonic parser.

## Source notes

- `path.go` embeds the grammar between the
  `BEGIN/END EMBEDDED path-grammar.jsonic` markers. That region is generated by
  the TS `embed-grammar.js` (run `npm run embed` from `ts/`) — do not edit it by
  hand.
- Differences from the TS port that are intentional, not parity bugs:
  - Each level's path is a **freshly allocated** `[]any` (Go does not pool/share
    arrays the way TS does), so callers do not have to copy before retaining it.
  - The plugin tags every alt it adds with the `path` group via
    `GrammarSetting{Rule: {Alt: {G: "path"}}}`, so path tracking can be excluded
    with `options.rule.exclude: "path"`.
- Path segments are `any`: map keys are `string`, array indices are `int`
  (deliberately `int`, not `float64`, so a type switch round-trips cleanly).
- `const Version` is bumped by the `publish-go` Makefile target.

## Debugging

```go
j := jsonic.Make()
j.Use(path.Path, nil)
j.Use(jsonic.Debug, map[string]any{"trace": true}) // log lex + rule steps
fmt.Println(jsonic.Describe(j))                     // dump tokens/rules/plugins
```

## Tests

`path_test.go` mirrors the TS suite. `TestLocalGrammar` is the parity of the TS
`expr` test: a small **non-JSON** integer-arithmetic grammar built with a local
`MatchValueSpec` value matcher, depending on nothing but the Jsonic parser. Keep
it aligned with the TS version.
